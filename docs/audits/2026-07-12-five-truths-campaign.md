## Constitution

editing are different truths. A lower framework concept is admitted only after
a reduced non-table witness proves that an existing owner cannot express the
required law. Similar appearance alone authorizes no merge.

Expanded is not a different table presentation tree. It is wrapping text
inside measured rows.

runs formatting, all-target compilation, the complete library suite, all three
example smokes, diff hygiene, and the protected `comparison_open: true`
witness. No campaign commit is pushed.

## Admission census

### Intrinsic sizing already exists

- `layout::intrinsic_height_for_width` asks a member for its requirement under
  a supplied cross-axis width. World text and text surfaces already report
  content-derived height.
- The table projection supplies authoritative resolved track widths. A row's
  content requirement is therefore the maximum child requirement at its track
  width plus one canonical envelope.
- No new "member knows its size" vocabulary is admitted.

### Retained variable extent owner is missing

- `virtual_list::variable::Region` already computes sparse measured heights,
  starts, total extent, visible range, and anchor correction.
- That geometry is carried inside `Materialization`, whose actual owner is
  existence (`range + pins`). `Materialization::eq` ignores its `Region`, while
  runtime retention is decided through that equality.
- This is a proven ownership contradiction. Checkpoint 1 may separate retained
  measurements from existence after a generic reduced witness fails.
- The uniform million-row implementation remains protected. It is rewritten
  only if a census proves duplicate ownership, not for aesthetic symmetry.

### Clip provenance exists but focus discards it

- Every frame already carries its inherited clip and paints beneath it.
- Focus outlines are collected as bare outlines and appended after the clip is
  popped. Late z-order currently erases provenance.
- Focus is the only proven deferred primitive with this defect. Checkpoint 2
  may repair that path; a generalized deferred-paint abstraction requires a
  second real caller.

### Focus modality and editing already exist

- `Focus::pointer()` and `Focus::keyboard()` already distinguish hidden and
  visible intent. `interaction::Tables` already owns the sole table edit
  identity. Text surfaces already distinguish read-only and editable modes.
- `shows_focus_indicator = visible || accepts_keyboard_input` incorrectly uses
  input ownership as a presentation proxy. Checkpoint 3 adds no modality or
  edit state; presentation must consume the owners already present.
- Deliberate double-click, F2, and Enter remain the default. Type-to-replace and
  edit-trigger policy are watch lines, not campaign work.

### Table presentation currently changes species

- Ordinary values branch between world-text constructors.
- Inactive editable cells branch between read-only TextBox and TextArea, giving
  one cell identity two focus, measure, paint, and selection paths.
- The legitimate presentation axes are line formation and residual overflow:
  Compact is single-line plus ellipsis; Expanded is wrapping plus clip.
- The legitimate extent policies are constant allocation and measured
  allocation with the configured row height as a floor.

## Required row law

For one unchanged row and cell recipe:

`requirement = max(cell intrinsic height at resolved track width) + envelope`

`Compact = configured row height`

`Expanded = max(configured row-height floor, requirement)`

Compact may allocate less than the requirement because its text policy owns
truncation. Expanded honors the requirement because wrapping exists to reveal
content. Virtualization records the resolved row extent; it never measures
table contents itself.

### Checkpoint 1 receipt

- `Materialization` now owns only existence: materialized range plus pins.
- A separate internal `Measurements` owner retains sparse measured extents;
  layout, traversal, runtime rebuilds, and departure cleanup carry it without
  widening the public provider contract.
- The reduced witnesses exposed two generic defects before any table repair:
  equality on materialization could discard changed measured geometry, and a
  variable sequence could require more than one projection/layout refinement
  after wrapped members changed extent.
- Presentation now uses a bounded fixed-point refinement for measured
  sequences. Uniform sequences remain on the existing constant-extent path.
- No table-specific sizing arithmetic and no public API were introduced.

### Checkpoint 2 receipt

- The defect reproduced outside tables: scene construction deferred a bare
  focus outline until after the frame's inherited clip had been popped.
- The internal deferred record now carries the already-resolved `Clip` beside
  its `Outline`; late paint replays clip, outline, and pop without changing or
  pre-clamping the outline rectangle.
- Base/chrome and popup-local scenes consume the same narrow focus record, so
  layer ownership remains where it already lived.
- The caller census found no second deferred primitive. Selection and other
  inline paint remain inline; no generalized deferred-paint system was added.

### Checkpoint 3 receipt

- `Focus` now answers only its existing visibility intent; focus kind and input
  capability no longer act as appearance proxies.
- `FieldMode` owns whether a projected text surface is editable. TextBox and
  TextArea retain editor chrome when pointer-focused only when that projected
  surface is editable.
- Table edit identity needs no new presentation state: projection already
  swaps the active cell to an editable surface, while at-rest selectable text
  remains read-only.
- Pointer-current display cells retain internal focus for selection, copying,
  keyboard continuation, and row activation without drawing a cell ring.
  Keyboard-current cells still request a restrained indicator.

### Checkpoint 4 receipt

- General world text now carries independent line-formation and residual-
  overflow axes. Existing single-line and wrapped constructors delegate to
  that one internal policy; no public API was added.
- Compact derives `(no wrap, configured ellipsis)` and Expanded derives
  `(word wrap, clip)`. Value cells use one constructor under those policies.
- Inactive editable cells use one read-only TextArea selection surface in both
  modes. Its glyphs come from the same world-text policy while its existing
  interaction layout supplies selection geometry. A TextBox exists only while
  the table's existing edit identity is active.
- Display TextAreas measure as text at the resolved track width, without the
  editor control-height minimum. The table retains one envelope and the
  variable sequence records the resulting row extent; Compact stays on its
  unchanged constant-extent path.
- Focused draft routing is generalized only for proven draft-owned surfaces.
  Generic document TextAreas remain application/document owned, a deliberate
  non-merge.
- Composition distinguishes node retirement from semantic-cell disappearance,
  preventing false draft pruning without weakening removed-node cleanup.
- Line clamp `(wrap, ellipsis)` is representable by the factored vocabulary but
  remains unimplemented until it has a caller.

- No replacement table track solver or duplicate text sizing path.
